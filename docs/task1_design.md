Task 1: Process Management and Threading — Design Document
1. Overview
The current assignment shows major concepts related to operating systems by means of C programs which use the POSIX threads library. Four programs were written to investigate such concepts as creating and managing threads, synchronizing processes with the help of mutexes, Round-Robin scheduling of CPUs and deadlock avoidance methods. Each of the programs is devoted to showing one of the discussed concepts through programming.
2. Thread Design
In this program, the basics of multithreading are shown by means of POSIX threads. The three threads are created via the pthread_create() function to perform different actions concurrently inside the program. Each of the threads works separately and prints messages which indicate their work. The pthread_join() function is used so that the main thread waited for all other threads to end their work before quitting. It is necessary to ensure that all the threads work properly.3. 3. Synchronization and Race Conditions
The second program mutex_sync.c illustrates the need for synchronization while several threads use the same data. Three threads increase a common counter by 1 million times which should produce 3,000,000 as a result. Without using synchronization, the actual output was 1,295,858. It happens due to the fact that the incrementation is not atomic operation; it comprises three parts: read current value, increment it and write to memory. In case several threads are doing it simultaneously, updates may interfere each other resulting in losing increments.
To fix this issue, a pthread mutex was added to the critical section containing an incrementing operation. Mutex guarantees that only one thread can work with a shared resource at a certain time period. After adding synchronization to the program, its output started to give 3,000,000 as expected.
4. Round-Robin Scheduling
In this simulation, we are trying to understand how the operating system assigns CPU time in Round-Robin scheduling among many processes. In Round-Robin Scheduling algorithm implementation, the time quantum is used and it defines the period for which each process executes until pre-emption takes place and the process is put at the end of ready queue. Different quanta have been used to see its effect on the behavior of the scheduling.
When the quantum was kept as one, there were many context switches and many scheduling overheads. This was because when quantum value was one, the processes were switched very frequently, allowing all the processes to get CPU time quickly and fairly. However, when the quantum was raised to 100, the context switches reduced and the processes had more chances to execute. The behavior was almost like First Come, First Served because of the reason that the processes finished their jobs before getting pre-empted by other processes.
5. Deadlock Prevention
Deadlock demo provides an example of how improper management of resources can lead to a permanent state of blocking within the system. In the case of unsafe version of the code, first thread obtains lock A and then tries to obtain lock B, whereas the second thread obtains lock B and after that tries to obtain lock A. Therefore, the condition of circular wait is created since each thread waits for the resource possessed by the other thread and the system becomes blocked because of deadlock.
To avoid this situation, the second version of the code is developed in such a way that the same order of acquiring locks is guaranteed for both threads. Since there is no condition of deadlock anymore, threads successfully finish their job and release the resources.
